# Refactor Plan: Consolidate Transaction Parsing Behavior

## Goal
Consolidate duplicated transaction parsing behavior across multiple files without caller-visible changes.

## Canonical Implementation

Create a new module `payment_gateway/transaction_parser.py` with the canonical transaction parsing logic:

```python
# payment_gateway/transaction_parser.py
def parse_transaction(raw: str) -> dict:
    """
    Parse transaction string into structured data.
    
    Args:
        raw (str): Transaction string in format "PREFIX|TXN_ID|CURRENCY|AMOUNT"
        
    Returns:
        dict: Parsed transaction with keys 'id', 'currency', 'amount'
        
    Raises:
        ValueError: If transaction format is invalid
    """
    parts = raw.split('|')
    if len(parts) != 4:
        raise ValueError('bad transaction')
    prefix, txn_id, currency, amount = parts
    if prefix not in ('TXN', 'PAY'):
        raise ValueError('bad prefix')
    return {'id': txn_id.strip(), 'currency': currency.upper(), 'amount': float(amount)}
```
