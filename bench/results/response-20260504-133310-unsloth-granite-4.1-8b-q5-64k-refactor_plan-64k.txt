### 1. Canonical Implementation  

Create a single, reusable function that encapsulates the duplicated parsing logic:

```python
# File: payment_gateway/transaction_parser.py
def parse_transaction(raw: str) -> dict:
    """
    Parse a raw transaction string into a normalized dictionary.

    Expected format: ``PREFIX|txn_id|currency|amount``

    Raises:
        ValueError: If the raw string does not contain exactly 4 parts,
                     or if the prefix is not 'TXN' or 'PAY'.
    """
    parts = raw.split('|')
    if len(parts) != 4:
        raise ValueError('bad transaction')

    prefix, txn_id, currency, amount = parts
    if prefix not in ('TXN', 'PAY'):
        raise ValueError('bad prefix')

    return {
        'id': txn_id.strip(),
        'currency': currency.upper(),
        'amount': float(amount)
    }
```

* **Why this is canonical**  
  * Handles the same validation and normalization steps used in every duplicated implementation.  
  * Centralizes error handling, making future changes (e.g., adding more prefixes) a single‑point update.
